Getting started

Install Mythodikal Anti-Virus and complete first-run setup on your machine.

  1. Download the installer for your platform (Windows, macOS, or Linux) from the Releases page.
  2. Run the installer. On Windows, double-click the .msi or .exe and follow the prompts. On macOS, open the .dmg, drag the app to Applications, then launch it. On Linux, install the provided package with your distribution's package manager.
  3. Windows — clear the SmartScreen prompt. Because Mythodikal ships unsigned (no paid OV/EV cert), Windows may show "Windows protected your PC." Click More info → Run anyway. If the file is blocked by Mark-of-the-Web, right-click the installer → Properties, tick Unblock, then re-launch.
  4. macOS — clear the Gatekeeper warning. Open System Settings → Privacy & Security, scroll to the notice for Mythodikal, and click Open Anyway.
  5. Grant any requested permissions. The app will ask for access to scan files on your disk. Approve the prompt so the engine can inspect files outside your home directory.
  6. Wait for initial setup. On first launch, Mythodikal loads its threat-definition database. This takes a few seconds and happens only once.
  7. You're ready. The main dashboard appears. From here you can start a scan, check your protection status, or adjust settings.
Tip: Mythodikal is entirely local-first — no account, no sign-in, and no data ever leaves your machine during setup or scanning.

Run a quick scan

A quick scan checks the locations most commonly targeted by malware — fast and low-impact.

  1. Open Mythodikal Anti-Virus from your Applications folder, Start menu, or app launcher.
  2. On the main dashboard, click Quick Scan.
  3. The scan begins immediately. A progress bar shows files scanned and elapsed time. You can continue using your computer normally during the scan.
  4. When the scan finishes, the results screen shows a summary: files checked, threats found, and time taken.
  5. If no threats were found, click Done to return to the dashboard.
  6. If threats were found, see Handle a detection for next steps.
Tip: Quick scans typically complete in under a minute and are a good daily habit after downloading files or visiting unfamiliar websites.

Run a full or custom scan

A full scan covers your entire disk. A custom scan lets you choose exactly which folders or drives to check.

  1. Open Mythodikal Anti-Virus and go to the Scan section.
  2. To scan your whole disk, click Full Scan. The engine will walk every accessible file. This may take several minutes depending on disk size.
  3. To scan a specific folder or drive, click Custom Scan, then click Choose Folder (or Choose Drive) and navigate to the location you want.
  4. Add multiple locations to a single scan by clicking Add another location and repeating the previous step.
  5. Click Start Scan. Progress is shown in real time — files scanned, current file path, and any detections as they occur.
  6. To pause, click Pause. Click Resume to continue. To stop early, click Cancel. Long scans persist their state across reboots — you can resume after a restart.
  7. When the scan completes, review the results summary. If threats were found, proceed to Handle a detection.
Tip: Custom scans are useful for checking a specific USB drive, downloaded archive, or shared folder without waiting for a full-disk scan.

Handle a detection

When Mythodikal flags a file as a threat, you can quarantine it, delete it permanently, or restore it if you believe it is safe.

  1. After a scan, the results screen lists each detected file with its name, location, threat classification, matched rule, and the detection source (feed name or YARA rule).
  2. To quarantine a file (recommended): select it and click Quarantine. The file is moved to secure, isolated storage where it cannot run. Your original file is preserved in case you need to restore it later.
  3. To delete a file permanently: select it and click Delete. Confirm when prompted. This action cannot be undone.
  4. To handle all detections at once: click Quarantine All or Delete All to apply the same action to every flagged file.
  5. To review quarantined files later: open the Quarantine section from the main navigation. Each item shows when it was detected and what threat it matched.
  6. To restore a quarantined file (if you are certain it is safe): select the file in Quarantine, click Restore, and confirm. The file is moved back to its original location.
  7. To permanently remove a quarantined file: select it in the Quarantine list and click Delete.
Tip: When in doubt, quarantine rather than delete. Quarantine keeps the file isolated but recoverable, so you can restore it if it turns out to be a false positive.

Turn on real-time protection

Real-time protection monitors your system continuously and flags threats the moment they appear — no manual scan needed.

  1. Open Mythodikal Anti-Virus and go to Settings (the gear icon or the Settings link in the main navigation).
  2. Find the Real-Time Protection section.
  3. Toggle the switch to On. The status indicator on the main dashboard changes from grey to green, confirming protection is active.
  4. If your OS shows a permissions prompt — for example, asking to allow file-access monitoring — click Allow.
  5. To turn off real-time protection temporarily (for example, while installing trusted software), return to this setting and toggle it Off. The dashboard will show a reminder so you don't forget to re-enable it.
  6. Toggle the switch back to On when you are done.
Note: Real-time protection is actively developed on Linux (fanotify daemon). macOS (FSEvents + opportunistic ESF NOTIFY) and Windows (ETW + AMSI + WDAC) real-time engines are part of upcoming releases — check the Releases page for the latest status.

Update threat definitions

Threat definitions are the database Mythodikal uses to recognise malware. Keeping them current ensures the latest threats are detected.

  1. Open Mythodikal and look at the Protection Status panel on the dashboard. The current definitions version and last-updated date are shown there.
  2. To check for an update immediately, click Check for Updates (or go to Settings → Threat Definitions → Update Now).
  3. If an update is available, the download begins automatically. A progress indicator shows download and installation status.
  4. When the update finishes, the dashboard shows the new definitions version and a confirmation message. No restart is required.
  5. To enable automatic updates, go to Settings → Threat Definitions and turn on Auto-Update. Mythodikal will check for and apply new definitions silently in the background.
  6. If an update fails (for example, due to no network connection), the app retries automatically. You can also retry manually by clicking Check for Updates again.
Tip: Mythodikal draws threat data from public feeds including MalwareBazaar and ThreatFox. Updates are incremental — only new or changed entries are downloaded, keeping the update small and fast.

Schedule scans

Set up recurring scans so Mythodikal checks your system automatically on a schedule you choose.

  1. Go to Settings → Scheduled Scans, or click the Schedule option in the Scan section.
  2. Click Add Schedule (or New Schedule).
  3. Choose the scan type: Quick Scan, Full Scan, or Custom Scan (with a folder you specify).
  4. Set the frequency: Daily, Weekly, or Monthly. For weekly, choose which day; for monthly, choose which date.
  5. Set the time the scan should start. Choosing a time when your machine is usually idle (such as overnight) avoids any impact on your work.
  6. Optionally, enable Run if missed so that if your machine is off at the scheduled time, the scan runs as soon as it next starts up.
  7. Click Save. The schedule appears in the list with its next run time shown.
  8. To edit or delete a schedule, select it from the list and click Edit or Remove.
Tip: A weekly full scan on a quiet day (such as Sunday night) combined with daily quick scans gives strong coverage without noticeable slowdowns during your workday.

Change settings

Mythodikal's settings let you tune protection, scan behaviour, notifications, and exclusions to fit how you work.

  1. Open Mythodikal and click the gear icon or the Settings link in the main navigation.
  2. Add a scan exclusion (a path Mythodikal should never scan): go to Exclusions, click Add Exclusion, and choose the path. Supports path, glob, hash, and signing-key exclusions. Exclusions are snapshotted per scan so history remains reproducible.
  3. Change notification preferences: go to Notifications and toggle which events trigger alerts — detections, completed scans, definition updates, or real-time protection status changes.
  4. Adjust scan performance: go to Scan Settings → Resource Usage and choose a throttle level. Lower settings reduce CPU and disk impact but make scans slower. The app automatically throttles on battery power.
  5. Change the quarantine storage location: go to Quarantine → Storage Path and click Change Folder. Pick a folder with enough space to hold isolated files.
  6. Reset all settings to defaults: scroll to the bottom of Settings and click Reset to Defaults. This does not affect quarantined files or scan history.
Tip: All settings are stored locally on your machine. Nothing is synced to a server or tied to an account.

Tips & troubleshooting

Common issues and how to resolve them.

  1. Scan seems slow or freezes. Open Settings → Scan Settings → Resource Usage and lower the throttle setting. On battery, the app throttles automatically. If the issue persists, try restarting the app.
  2. A file was quarantined that you know is safe (false positive). Go to the Quarantine section, select the file, and click Restore. Then add its containing folder to Settings → Exclusions to prevent future alerts.
  3. Threat definitions appear out of date. Go to Settings → Threat Definitions and click Update Now. Check your internet connection if the download fails.
  4. Real-time protection keeps turning off. Some operating systems require the app to have persistent background access. Check System Settings → Privacy & Security (macOS) or Windows Security settings to ensure Mythodikal is allowed to run in the background.
  5. The app shows a permissions error on macOS. Open System Settings → Privacy & Security → Full Disk Access and make sure Mythodikal Anti-Virus is listed and enabled.
  6. A scheduled scan did not run. Check that Run if missed is enabled in the schedule. Also confirm the machine was on at the scheduled time, or enable run-if-missed so it catches up on next launch.
  7. Need help or found a bug? Open an issue on the GitHub repository. Include your OS version and a description of what you expected vs. what happened.

Privacy & local-first

Mythodikal is designed from the ground up to protect your privacy. Here is what that means in practice.

  1. All scanning happens on your device. Files you scan are never uploaded, streamed, or sent to any server. The engine runs entirely locally.
  2. No account required. There is no sign-in, registration, or cloud account. You install the app and use it — nothing more.
  3. No telemetry. Mythodikal does not collect usage data, crash reports, or diagnostic information. Nothing is phoned home.
  4. Threat definition updates are one-way. When definitions update, your machine downloads new data from the update server. No information about your system or scans is sent in return.
  5. All data stays on your machine. Scan history, quarantine contents, settings, and logs are stored in local application data and are never synchronised anywhere.
  6. To review what data the app stores locally, go to Settings → Privacy. From there you can also clear scan history or the quarantine if you wish.
